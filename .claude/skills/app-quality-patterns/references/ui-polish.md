# QP — UI polish (loaded on trigger match; see ../SKILL.md)

| ID | Trigger | Requirement | Verify (copy-ready smoke recipe) | Anti-pattern |
|---|---|---|---|---|
| QP-20 | Dropdowns, selects, accordions, expandable rows | **Indicator icons align and animate truthfully.** Chevrons/carets are vertically centered against their label's line-height, share one size/weight across the app, and their rotation matches the open/closed state. | Open the dropdown/accordion → screenshot or inspect → chevron centered (not riding high/low vs text), rotated state correct; collapsed state restores it. | Chevron baseline-aligned (sits low), differing sizes per component, rotation stuck after close — the classic "always slightly off" arrow. |
| QP-21 | Every list, table, detail, or data-driven panel | **All three non-happy states are designed: empty, loading, error.** Empty state says what this is and how to add the first item; loading is bounded (skeleton/spinner with timeout); error is actionable (retry / support path). | Load the view with 0 records → assert designed empty state (not blank/white); throttle to observe loading state; force a failed fetch → assert actionable error. | Blank white div on empty fetch; infinite spinner on error; raw exception text shown to the user. |
| QP-22 | Any form | **Validation UX preserves the user's work.** Errors render inline at the field, submit never wipes entered values, focus moves to the first invalid field, and the submit error summary (if any) links to fields. | Fill a form with one invalid field → submit → assert inline error at the field, all other values intact, focus on the invalid field. | Top-of-page generic "form invalid"; cleared form on failed submit; error visible only after scrolling. |
| QP-23 | Any timestamp or date display/input | **Times render correctly in the user's zone, always.** Date-only values never shift across midnight boundaries; relative times ("2h ago") have an absolute tooltip/fallback; the standard means is UTC storage + locale-aware rendering with a deliberate format. | Create a record at a near-midnight time in a non-UTC zone → assert the displayed date matches the user's expectation in their zone, and the stored value is UTC. | Date shows "yesterday" for some users; `toString()` server-zone leaks; date-only field saved as midnight UTC shifting a day. |
| QP-24 | Any form or editor with unsaved state | **Unsaved changes are guarded.** Navigating away (in-app or browser) from a dirty form prompts to save/discard — work is never silently lost; drafts persist where the flow expects it. | Edit a form without saving → navigate to another view and trigger browser back/close → assert a save/discard prompt (or auto-draft restored on return). | Route change silently discards a half-written record; refresh loses an hour of input. |
