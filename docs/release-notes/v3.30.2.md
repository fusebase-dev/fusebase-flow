## Changed

### Bug A — pre-commit secret-scan self-trips on `secret-patterns.yml`

`hooks/git/pre-commit` fed the whole `git diff --cached -U0` (including removed `-` lines) to `secret_scanner.scan()`, so a Flow upgrade that edited `policies/secret-patterns.yml` tripped the scanner's own designed example tokens and **BLOCKED the commit**. The obvious `whitelist:` escape broke fixtures 10/11.

- **Fix (`hooks/shared/staged_secret_scan.py` + `hooks/git/pre-commit`).** Scan only **added (`+`) lines**, and **path-exclude** the scanner's own designed-token files — `policies/secret-patterns.yml`, `policies/secret-patterns.local.yml`, and `hooks/tests/fixtures/` — via a Python helper. `secret_scanner.scan()` is **unchanged** (no semantic change). **No whitelist needed** (avoids the fixture-breaking trap). Independently corroborated by two consumers (one wrote the same path-exclusion locally).
- **Result:** editing `secret-patterns.yml` no longer blocks; a real secret on a `+` line in a normal file **still blocks**; fixtures 10/11 still detect. The excluded-file gap is deliberate and documented; the misleading "just whitelist it" guidance is stripped from the pre-commit messages, and the PreToolUse self-trip when an agent *writes* full `secret-patterns.yml` content is documented as a known limitation.

### Bug B — MSYS bounded-run hang (B1) + health-check false-BROKEN (B2)

Root cause (corrected by consumer field evidence, which overturned the first review): on MINGW64, `run_with_timeout` = `timeout -k` reaps only the **direct child**. An orphaned, **pipe-holding native grandchild** survives POSIX timeout cleanup, so any bounded op **captured via `$(…)`** never receives EOF on the pipe the orphan still holds → the parent's command-substitution **blocks past the deadline** → CPU-bound hang + accumulating runaway `bash.exe` (B1). The same un-reaped run surfaces an rc `ffhc_timed_out` doesn't recognize → non-deterministic false **BROKEN** (B2). B1 and B2 share **one** root cause.

- **Tempfile capture (belt #1, all platforms — the guaranteed anti-hang).** At the bounded `$(…)` capture sites — `ffhc_run_bounded` and the conflict reporter — the bounded process's stdout/stderr is redirected to a TEMP FILE, then `wait`ed and read. The parent's `$(…)` can no longer starve on a pipe a descendant still holds, even if a tree-kill races or misses. Robust tempfile handling (explicit `${TMPDIR}/ffhc-bounded.$$.XXXXXX` template; a tempfile failure routes to SKIPPED → UNVERIFIED, **never** a false BROKEN or a hang; no transient files in CWD).
- **MSYS best-effort process-tree kill (belt #2, MINGW*/MSYS*/CYGWIN* only).** The bounded command's Windows pid is captured **at launch** (`/proc/<pid>/winpid`); on timeout, `taskkill //F //T //PID <winpid>` (output suppressed) reaps a native/escaped descendant tree. Guarded strictly, falls back gracefully if `/proc/<pid>/winpid` or `taskkill` is absent (never hangs, never errors the bounded contract). **Honest limit:** this is **best-effort** (Windows does not reparent orphans deterministically) — the tempfile capture is the guaranteed anti-hang; the tree-kill is the runaway reaper.
- **POSIX (Linux/macOS): UNCHANGED.** `run_with_timeout` is byte-identical to v3.30.1 (`timeout -k` preserved — lower regression risk for the 26 timeout tests). `124` (deadline) / `137` (force-after-grace) preserved.
- **B2 defense (deterministic + a belt).** With the core fix, a cleanly-reaped timeout now surfaces `124` → `ffhc_timed_out` → `PARTIAL_UNVERIFIED`, not BROKEN. As a belt for any residual non-124 signal rc, the health-check reclassifies a hook-test run with **no `FAIL:` + no strict `N/N PASS` + (`rc==124 || rc>=128`)** to advisory `HOOK_TESTS_INCONCLUSIVE` → `PARTIAL_UNVERIFIED` (exit 4). A **no-strict-PASS guard** keeps a crash-after-PASS reading BROKEN; a genuine crash (`rc 1..123/125..127`, or malformed `rc==0`) STAYS BROKEN (exit 2).
- **Windows escape (B3).** `--skip-hook-tests` alias for `--fast` (which becomes reliable once B-core lands — today `--fast` runs the conflict reporter's bounded `$(…)` and intermittently hangs).
- **Upgrade observability (B4).** Progress echoes bracket `upgrade.sh`'s previously-silenced Step 2 re-mirror.
