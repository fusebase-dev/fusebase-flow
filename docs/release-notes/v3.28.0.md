# FuseBase Flow v3.28.0 — FR-27 liveness discipline (anti-hang)

**Release date:** 2026-06-17
**Previous version:** 3.27.0
**Type:** MINOR (feature — additive)

## Summary

**FR-27 — liveness discipline** is the first always-on rule added since FR-26. Driver: a real consumer hang — a project's AI developer launched a background re-verify probe that **HUNG** (no internal timeout; a fetch/cleanup stalled against a cold-start proxy). A hung process emits **no completion event**, so the agent was never re-invoked and idled silently until the operator nudged it ("is it all done?").

The framework already named this failure — but only for **delegated sub-agents** (`task-delegation` turn-completion / `BLOCKED-AT`, scope-gated). The agent's **own** probe / script / deploy / fetch-loop / browser-automation was uncovered, and the protocol had been hand-retyped into 3 deploy handoffs + the `subagent-deploy-run-synchronously` memory with no canonical home.

FR-27 ships the rule + a new **32nd canonical skill** (`liveness-discipline`) + the structural **`hooks/local/lib/bounded-run.sh`** watchdog helper + **3-tier present-by-construction delivery**. Additive and **FR-07-clean**: FR-27 is an APPEND to the FR table — existing FR-01..FR-26 rows are byte-unchanged, and the 3 deploy-policy semantics + `ratchet-governance.yml` + `run-with-timeout.sh` (`ffhc_*` API) are untouched. Double-reviewed by Codex (design RESCOPE folded + implementation review: a stale-count DO-NOT-SHIP → fixed → re-confirm **SHIP**).

## The honest enforcement model (load-bearing)

**No hook can reliably verify this hang class** — there is no elapsed-time / idle event in the hook schema, `post_tool_use` cannot fire for a call that never returns, and a Stop-time "watchdog: applied" signal would be **attestation theatre** (present-or-absent independent of real behavior — the inert-lever anti-pattern). Therefore FR-27 ships **no blocking gate and no verification hook** (the FR-26 precedent: a real hang vs a legitimate long-but-live run is semantic; a gate would train premature kills).

Enforcement = **(1) structural safe-by-default tooling** (the `bounded-run.sh` wrapper bounds a *monitored* background launch to a wall-clock deadline so it reaches completion-or-death instead of a 0-byte silent idle) + **(2) present-by-construction delivery** (the rule is in context at the moment work would be launched).

**Qualified tooling claim (do not overstate):** the wrapper bounds the **monitored** process only. It does **not** kill an `&`-detached grandchild or an uninterruptible OS wait, and does **not** prove the host re-invokes the agent. The skill teaches: don't `&`-detach under the wrapper; put a deadline INSIDE long scripts too; always flush partial results.

## What ships

| Piece | What it does |
|---|---|
| **FR-27 rule** (`FLOW_RULES.md`) | New FR-table row (append only) + amendment-log entry: any long/silent background work must be made observable BEFORE launch — bounded by a timeout/watchdog, OR completed in-turn, OR returned as `BLOCKED-AT-<gate>` + a record-then-read pointer. A task that cannot signal its own completion-or-death must never be launched bare. |
| **`liveness-discipline` skill** (`flow-skills/liveness-discipline/SKILL.md`) | 32nd canonical skill — the full protocol (never launch bare → attach ≥1 liveness guarantee; diagnose a suspected hang by activity/mtime, not 0-byte existence; on-confirmed-hang recovery) + a compact `bounded-run` example + cross-links to `task-delegation` (`BLOCKED-AT`) and `smoke-testing` (record-then-read). Clean-room note present. |
| **`bounded-run.sh` structural helper** (`hooks/local/lib/bounded-run.sh`) | **Reuses** `run-with-timeout.sh`'s core (timeout/gtimeout detection, `-k` kill-grace, rc-124/137 classification, no-binary skip policy) WITHOUT renaming or breaking the `ffhc_*` API the health-check sources. Adds a wall-clock **deadline** that emits a terminal **timeout line** + **incremental heartbeat** logging so a bounded job reaches completion/death (not a silent idle). |
| **3-tier present-by-construction delivery** | **Tier 1:** one FR-27 pointer row in the role-discipline § Write-time discipline digest (proven FR-26 channel, all tool-using execution). **Tier 2:** a liveness-reminder line in the `session_start.py` bootstrap string. **Tier 3:** the liveness/`BLOCKED-AT` clause promoted UP into the handoff Role-bootstrap **Hard-invariants** (so the MAIN session carries it, not just the parallel-Tracks section). |
