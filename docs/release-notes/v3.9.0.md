# Fusebase Flow v3.9.0 — canonical skills relocated to `flow-skills/`

**Release date:** 2026-06-04
**Previous version:** 3.8.7

## Summary

Resolves the long-standing **U12** collision: the FuseBase CLI deprecates the root `./skills` folder (`⚠️ The ./skills folder is obsolete and should be deleted`), which Flow had used as its **canonical** source of truth. The v3.8.3 guard made "keep root `skills/`" safe (non-foreclosing); this release picks the durable end-state — relocate Flow's canonical store to a Flow-namespaced path the CLI never touches — so the collision cannot exist under any CLI behavior, current or future.

This is a structural change to Flow's mirror/upgrade/health spine, shipped with zero-touch automatic migration for existing installs.

## Changed

### Canonical skills now live in `flow-skills/` (was root `skills/`)

- **Why `flow-skills/` and not `.fusebase-flow/skills/`?** `.fusebase-flow/` is already gitignored as a defensive runtime-state namespace (`.gitignore`); tracking canonical source inside it would need a fragile ignore-exception and conflate "source" with "runtime state". A visible, tracked top-level `flow-skills/` is unambiguous, and the CLI deprecation targets the literal `skills` name only — so `flow-skills/` is collision-free.
- **Scope: skills only.** `agents/` is not deprecated by the CLI and stays at root.
- **Provider mirrors unchanged.** `.claude/skills/<name>/SKILL.md` and `.agents/skills/<name>/SKILL.md` keep their paths (generated by skill name). The CLI still owns `.claude/skills/`; Flow still regenerates it from canonical. `.claude-plugin/plugin.json` is unaffected.

### Zero-touch migration on upgrade

`bash hooks/local/upgrade.sh` (and `bootstrap-upgrade.sh`, which hands off to it):

1. Lands canonical at `flow-skills/` from the upstream source (`CONTENT_DIRS`).
2. Retires a legacy root `skills/` — backed up as `skills.pre-upgrade-<ts>`, then removed.
3. Re-mirrors `flow-skills/` → providers.

Idempotent (a second run is a no-op). Every reader prefers `flow-skills/` and falls back to legacy `skills/`, so a partially-migrated or pre-upgrade tree keeps working. Fresh installs: the install docs now copy `flow-skills/`.

### Readers repointed

`mirror-skills.sh`, `fusebase-flow-health-check.sh`, `check-cli-flow-conflicts.sh`, `preflight.sh`, `upgrade.sh`, `sync-version-strings.sh` (skill-count), `session_start.py`, `command_policy.py`, `upgrade-engine.sh`. The CI public-surface allowlist accepts `flow-skills` and rejects a stray top-level `skills/` reappearing.

### Health guard inverted (not removed)

- Canonical absent (`flow-skills/`) while Flow mirrors exist → loud, recoverable **`FLOW_LAYER_DRIFT`** naming the restore path (`upgrade.sh` / `bootstrap-upgrade.sh` / `git checkout -- flow-skills/`).
- A leftover legacy root `skills/` alongside `flow-skills/` → **benign INFO** advising the idempotent migration. The CLI's "delete ./skills" warning is now correct for Flow too, so it's safe to follow.

### Docs

README/AGENTS overlay "do NOT delete root `skills/`" guard note replaced with the relocation explainer; install guides, edition-boundary docs, workflows, templates, and policies repointed to `flow-skills/`.

## Upgrade notes

- Existing installs: run `bash hooks/local/upgrade.sh` (refresh `.fusebase-flow-source/` first). It migrates `skills/` → `flow-skills/` automatically and retires the old dir (backup kept). After validating, you may delete the `skills.pre-upgrade-*` backup.
- Rollback: `git revert <SHA>` (the relocation is one commit). Migrated downstreams keep the `skills.pre-upgrade-*` backup.
