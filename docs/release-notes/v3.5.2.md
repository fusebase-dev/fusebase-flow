# Fusebase Flow v3.5.2 — recovery/overlay refresh

**Release date:** 2026-05-31
**Previous version:** 3.5.1

## Summary

A health-and-recovery audit found the recovery **overlay templates** (what `post-fusebase-update.sh` writes back into a downstream project after a `fusebase update` wipe) had not kept pace with the v3.3–v3.5 additions. This repo's health was already HEALTHY; the staleness was latent — it would only surface in a downstream project that ran recovery. v3.5.2 fixes it. No skills added or removed (still 23 / 46 manifest lines).

## Fixed

### R-1 — recovery now restores all slash commands

`post-fusebase-update.sh` Step 8 restored only `fusebase-health.md`. It now loops over **all** `hooks/local/fusebase-flow-overlays/commands/*.md`, so `/onboard` and `/product-owner` survive a CLI refresh. Their templates were added to the overlay. Future commands are covered automatically (no hardcoded names). Verified by simulating a command wipe: recovery restored 2 of 3 (skipping the unchanged one), bytes identical to committed.

### R-2 — overlay skills lists + discovery instruction refreshed

The AGENTS.md and CLAUDE.md overlay templates listed only the original on-demand skills. Both now enumerate all 23 canonical skills and include the **"Active project context"** discovery instruction (read `docs/north-star.md` / `docs/<app>/product.md` if present; never auto-create), so a downstream that recovers gets an accurate, hook-independent overlay.

### R-3 — CLAUDE.md overlay FR/version labels

The CLAUDE.md overlay self-attestation and skill list still referenced `FR-19` / `v3.1`. Swept to `FR-20` / `v3.5.0`.
