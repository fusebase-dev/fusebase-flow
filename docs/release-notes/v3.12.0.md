# Fusebase Flow v3.12.0 — FR-23 documentation budget

**Release date:** 2026-06-07
**Previous version:** 3.11.1

## Summary

A new always-on rule, **FR-23 (documentation budget)**, plus the skill that operationalizes it. AI-consumed artifacts (spec, decisions, tasks, gate, handoff, product/business-logic docs, project-internal skills) are created only when they reduce future context cost more than they add. Duplicate rationale, narrative padding, and docs created merely because a template exists are paid in tokens on every future load and spawn stale conflicting copies. FR-23 is the documentation-axis complement to FR-21 (ceremony proportional to change size): FR-21 scales *process*, FR-23 scales *persistent documentation*. Tier 1 == the Lightweight change-note.

## Design choices

- **Active vs formal handoff split.** Active session continuity is a single superseded `docs/tmp/handoff.md` (FR-18 / Tier 2); formal implement/deploy role-relay prompts stay at `docs/handoff/*`. New content never routes active continuity through `docs/handoff/`.
- **Business logic: AI-index by default, human narrative preserved.** A new retrieval-index template is the AI default; the existing human-narrative template is retained for explicit human-readable requests rather than overwritten.
- **No new safety surface.** FR-23 governs *whether/where* documentation exists, not gates. Full lane and the approval-gated side effects (FR-05 / FR-07 / FR-12) are unchanged.
