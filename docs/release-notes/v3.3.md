# Fusebase Flow v3.3.0 — generic flow skills + FR-20

**Release date:** 2026-05-31
**Previous version:** 3.2.0

## Summary

Gap analysis of the FuseBase positioning source identified AI-development failure modes the framework did not yet address. v3.3.0 ships the subset that needs **no user input** (generic engines, deliverable now): an always-on anti-patch rule plus four skill additions/extensions. The input-dependent skills (north-star, client-vs-internal, product-docs-first, business-logic-guardian, product→apps) are deferred to the onboarding keystone slice, since they only work once a project has provided its vision.

## Added

### FR-20 — Zoom out, don't patch-myopically

New always-on rule: before committing a fix or improvement, zoom out and confirm it targets the root cause, stays consistent with spec / decisions / North Star (if present), sits in the right layer, and won't create drift elsewhere. Patch-on-patch accumulation is a primary driver of AI-development drift. Paired with FR-10 (reproduce-before-fix). Self-attestation updated to FR-01..FR-20.

### `zoom-out` skill

Operationalizes FR-20: root-cause-vs-symptom, layer, consistency, drift, and repeat-patch checks before a fix is committed.

### `phase-audit` skill

Spawns an independent sub-agent to audit ALL slices of a completed phase against the spec, plus cross-slice consistency — catching drift that per-slice in-session review misses. Composes `task-delegation` + `code-review`.

### `git-history-diagnostic` skill

Regression archaeology: locate where a regression entered by comparing/bisecting commits, so the fix targets the actual causing change. Read-only diagnosis; hands the fix to `zoom-out` + `validation-and-qa`.

## Compatibility

Existing projects adopt v3.3.0 by pulling the updated framework files and regenerating mirrors:

```bash
bash hooks/local/mirror-skills.sh
bash hooks/local/preflight.sh
bash hooks/tests/run-tests.sh
```

No per-ticket artifact migration is required.
