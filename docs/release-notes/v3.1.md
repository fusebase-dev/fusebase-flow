# Fusebase Flow v3.1 — FR-19 chat-text questions, no popup menus

**Release date:** 2026-05-27
**Previous version:** 2.9.0

## Summary

v3.1 makes operator questions transcript-first. Agents must write clarify prompts, option choices, deploy confirmations, and recovery decisions as normal chat text instead of clickable popup / modal menus.

This Fusebase CLI edition also packages Fusebase Apps CLI provider assets alongside the Flow lifecycle framework. Flow remains the source of truth for specs, decisions, tasks, verification gates, reviews, deploy handoffs, and smoke discipline. CLI provider skills, app agents, and Claude Code quality hooks provide Fusebase Apps runtime/domain guidance.

## Added

### FR-19 — Chat-text questions, no popup menus

New always-on rule:

> Operator questions and decision prompts must be written as normal chat text, usually a short options table or numbered list. Do not use modal popup / clickable menu tools (`AskUserQuestion` or equivalents) because they cannot be copied, forwarded, quoted, or followed up on reliably across sessions.

### `smoke-testing` skill

New role-linked skill for outcome-based smoke discipline:

- Product Owner uses it to define S1..Sn with operator-visible success criteria, ground-truth diagnostics, adversarial checks, and evidence requirements.
- AI Developer / Deploy phase uses it to execute smoke and cannot claim PASS from exit code, hash, service status, symbol presence, or auth sanity alone.
- If end-to-end smoke is not feasible, the deploy report must say `PENDING-OPERATOR-SMOKE`; spec stays DRAFT.

### `design-discovery-ideation` skill

New PO-owned skill for divergent product/UI/workflow option discovery:

- Product Owner uses it when the operator asks for options, variations, alternatives, or other possible shapes.
- It separates fixed constraints from open design dimensions, then presents 2-4 meaningfully different options in chat text.
- For frontend/UI tickets, it now emits an implementation-ready design brief with product identity, surface map, data/API contract, applicable stack conventions, stable selector strategy, trust-critical interactions, and explicit non-goals.
- AI Developer must not use it to invent new product direction after decisions lock; implementation follows the selected direction or stops on conflict.

### `task-delegation` skill

New role-linked skill for bounded delegation:

- Product Owner may delegate read-only investigation, option comparison, artifact review, or doc-only drafting.
- AI Developer may delegate independent implementation/test slices only with disjoint ownership and main-session integration.
- Deploy phase may delegate read-only triage only; deploy, rollback, approval, secrets, and live-session smoke stay in the main Deploy phase session.

### `skill-authoring` skill

New role-linked skill for clean-room reusable skill creation:

- Product Owner uses it to classify framework skill vs project skill vs problem-catalog entry, compare overlap, and define role applicability before implementation.
- AI Developer uses it when implementing approved framework skill edits: canonical-first, mirror refresh, manifest refresh, source-leak scan, stale-count scan.
- External skill material is treated as concept-only; copied prose, examples, proprietary labels, and vendor-specific paths are forbidden.

### Fusebase CLI edition packaging

New edition-specific assets and docs:

- Added `docs/fusebase-cli-edition.md` as the Flow/CLI boundary map and overlap table.
- Added 19 Fusebase Apps CLI provider skills to `.claude/skills/` and `.agents/skills/`.
- Added CLI app agents `app-architect` and `app-create-checker` to `.claude/agents/` and `.codex/agents/`.
- Added CLI Claude Code quality hooks under `.claude/hooks/`.
- Updated `.claude/settings.json.example` to keep CLI Stop hooks and Flow lifecycle hooks in one additive example.
- Updated clean-room/source-map docs to distinguish canonical Flow files from copied CLI provider assets.
- Updated health check behavior so source-template / edition projects validate without requiring downstream overlay markers.

## Rationale

Operators reported that clickable popup menus are inconvenient in the Fusebase Flow relay loop:

- Options cannot be copied into another session.
- Popup state is hard to scroll back to after context changes.
- Follow-up questions are awkward because the UI expects a click.
- Audit wording is lost because the question is not durable chat text.

v2.7.1 fixed this for Product Owner only. v3.1 applies the same principle framework-wide.

## Compatibility

Existing projects can adopt v3.1 by pulling the updated framework files and regenerating mirrors. This release now contains 14 canonical Flow skills after adding design-discovery-ideation, smoke-testing, task-delegation, and skill-authoring:

```bash
bash hooks/local/mirror-skills.sh
bash hooks/local/mirror-agents.sh
bash hooks/local/preflight.sh
bash hooks/tests/run-tests.sh
```

No per-ticket artifact migration is required.

For the Fusebase CLI edition, provider surfaces contain 33 skills total: 14 Flow mirrors plus 19 CLI provider skills. The Flow mirror manifest tracks only the 14 canonical Flow skills.
