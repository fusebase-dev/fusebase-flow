# Fusebase Flow v4.2.0 — hook-layer manifest verification + single-process test runner

**Windows/Git-Bash operators can now reach a full `HEALTHY` health-check verdict.** Before v4.2.0 the health check's hook-test critical ran a fork-per-case bash suite; MSYS process creation is ~100× slower than Linux, so on Windows the full run hung and the verdict was capped at `PARTIAL_UNVERIFIED` (exit 4) forever. This release replaces that critical with a fast, OS-independent content-hash **manifest verify**, and makes the optional local test suite MSYS-viable.

## Highlights

- **Manifest-verify critical.** The hook-test critical is now a single-pass hash compare of the Flow-owned hook layer against a committed, byte-stable manifest (`audit/hook-layer-manifest.json` — no timestamps; a pure function of covered bytes + `VERSION`). Full `HEALTHY` / exit 0 is reachable on stock Win11 + Git-Bash in **< 60 s** (measured ~31 s). A mismatch names the exact drifted file (`FLOW_LAYER_DRIFT`, exit 1) and **also catches local tampering** that the old "re-run the tests" approach could not.
- **Single-process fixture runner.** `hooks/tests/run_hook_tests.py` drives each handler's `main()` in-process (same 21 fixtures + assertions); `--compare-subprocess` proves in-process ≡ subprocess for the `(exit_code, decision, rule_id)` triple (21/21, a permanent CI step). The MSYS fixture phase drops from minutes to ~0 s. FR-07-protected handlers/shared are unmodified.
- **In-repo release gate.** `fusebase-flow-verify.yml` is now reusable (`workflow_call`) with runner-parity + manifest-freshness steps; the new `fusebase-flow-release.yml` creates a GitHub Release only via a `publish` job gated `needs: verify` — a red suite ⇒ no Release for that tag.
- **Platform-adaptive `--run-hook-tests`.** Optional deep diagnostic (never gates the verdict): full `run-tests.sh` on POSIX; on MSYS a fast subset (single-process fixtures + git-smoke + hook-manifest) completing **< 120 s** (measured ~52 s). `--run-hook-tests-full` / `FFHC_RUN_HOOK_TESTS_FULL=1` forces the full suite on MSYS.
- **Upgrade propagation.** `upgrade.sh` carries the manifest so existing consumers pick up integrity verification with no manual step.

## Trust model

The manifest proves *"this install byte-matches release 4.2.0"*; upstream CI (Linux) proves *"release 4.2.0 passes its full hook-test suite."* Consumers verify membership (fast, any OS) rather than re-deriving correctness.

## Compatibility

Fully backwards compatible: `--fast` still returns exit 4 (now unnecessary on Windows — kept for back-compat); verdict names and the exit map (0/1/2/3/4) are unchanged; Linux/macOS behavior and the default full `run-tests.sh` coverage are unchanged. No new consumer dependencies (bash, python3, git only).
