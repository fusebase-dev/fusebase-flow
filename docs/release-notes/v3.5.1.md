# Fusebase Flow v3.5.1 — post-implementation audit corrections

**Release date:** 2026-05-31
**Previous version:** 3.5.0

## Summary

An independent read-only audit of the v3.3.0–v3.5.0 work surfaced one integrity defect and a cluster of consistency-staleness issues. v3.5.1 fixes them. No new capabilities; no skills added or removed (still 23 canonical / 46 manifest lines).

## Fixed

### Claimed-but-missing skill extensions (HIGH)

The v3.3.0 release notes / spec asserted two skill extensions that were never actually written. Now implemented for real:

- **`skill-authoring` → "Domain-expert skill mode"** — how to author a project-local industry-expert skill (market-aware, clean-room, no competitor names, not mirrored).
- **`design-discovery-ideation` → "Prototype before build"** — ASCII mockup / optional throwaway HTML screen → operator feedback → build.

This also resolves two dead cross-references: `product-docs-first` and `project-onboarding` pointed at "skill-authoring (domain-expert mode)", which now exists.

### FR-20 consistency sweep (MEDIUM)

A fresh agent self-attesting from satellite files (GEMINI.md, CLAUDE.md, copilot-instructions, agent files, workflows, handoff templates, role-discipline) was pledging "FR-01 through FR-19" — omitting FR-20 (shipped v3.3.0). Swept `FR-01..FR-19` → `FR-01..FR-20` and stale `v3.1`/`v3.2.0` labels → `v3.5.0` across ~32 non-historical files. Dated release notes and handoffs left intact (historical record). Legitimate mentions of the FR-19 *rule itself* (no-popup-menus) preserved.

### README (LOW)

- Corrected stale skill counts: 14 → 23 canonical; added the 9 v3.3–v3.5 skills to the catalog table.
- Version badge → 3.5.1.

### session_start.py glob (LOW)

Project-artifact scan now uses `rglob` so nested app layouts (`docs/apps/<app>/product.md`) are surfaced, not only single-level `docs/<app>/`.
