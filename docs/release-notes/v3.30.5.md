# FuseBase Flow v3.30.5 — hook-security hardening (FR-07 + FR-12 fail closed at every load-point)

**Release date:** 2026-07-03
**Previous version:** 3.30.4
**Type:** PATCH (security hardening — fixes a SHIPPED FR-07 protected-path bypass)

## Summary

Hardens both pre-commit security controls so they **FAIL CLOSED at every reachable load-point**: the FR-07 §3 protected-path guard AND the FR-12 §2 secret scanner. After this release, no mutable working-tree Python — code, patterns, startup file, env var, or import path — can influence any security check. The umbrella lesson: a security control that loads or executes code, patterns, or config from the working tree can be neutralized by tampering that working-tree copy; the fix runs security checks from the TRUSTED committed HEAD copy.

Found by the whole-roadmap review as a still-SHIPPED bypass. No new FR rule (FR-01..FR-27 unchanged) and no new Flow skill (32 unchanged) — the enforcement code is hardened, not the rule set. Converged over TEN adversarial rounds (each closed a distinct, named mechanism forming a finite ladder of "how working-tree code reaches the interpreter"); the final deploy-gate confirm was SHIP from two independent reviewers (a Codex companion with zero findings + a 3-lens Opus panel with RED→GREEN PoCs). FR-07-clean: no FR rule rows changed, no deploy-policy rule semantics, no `ratchet-governance.yml` touched — only the FLOW_RULES version-attestation line moved.

## What ships

| Piece | What it does |
|---|---|
| **Staged delete / rename coverage** | The protected-path check now covers staged DELETES (`git rm`) and RENAMES of protected files (was `--diff-filter=ACM` only), via `path_policy.staged_change_paths` (A/C/M→path, D→deleted, R→old+new). The single-use digest-bound exception is extended to delete/rename, never weakened. |
| **Import / enumeration / exception fail-opens closed** | Import error (e.g. missing PyYAML) ⇒ BLOCK with an FR-07 diagnostic (was a silent fail-open); missing python3 ⇒ loud WARN; enumeration failure ⇒ BLOCK (was a silent []-pass); `BaseException`/`SystemExit(0)` from a tampered module ⇒ BLOCK; `git name-status` nonzero rc ⇒ raise/BLOCK (no partial-list bypass). |
| **Policy-present + additive-only override** | Missing/empty/malformed `protected-paths.yml` ⇒ BLOCK (was a total FR-07 disable); `protected-paths.local.yml` can no longer erase or relax protected categories (additive-only). |
| **Trusted-HEAD enforcement** | FR-07 (and, in `-S`, the FR-12 scanner + its patterns) run from a TRUSTED committed HEAD copy extracted into a fresh `mktemp` dir, under `python3 -S` with a scrubbed env (`unset PYTHONPATH/PYTHONSTARTUP/PYTHONHOME`, `PYTHONNOUSERSITE=1`), and the trusted-vs-fallback decision is made by git (unforgeable). Closes staged/unstaged tamper, `sitecustomize.py`/`usercustomize.py` startup injection, a hostile `PYTHONPATH`, and the deepest tail — a repo-root `pathlib.py`/`yaml.py` shadowing a stdlib/site import when CWD is on `sys.path[0]` under `python3 -S -`. Fixed by running the §2/§3 MAIN checks as trusted FILE SCRIPTS from the temp dir (CWD never added), plus `PYTHONSAFEPATH=1` and prepend-site-packages. Also converts the `$(git ls-tree)` sentinel loops to file-redirect to end an intermittent MSYS command-substitution hang. |
| **Hook-install call-site rc** | No silent "installed" on failure. |
