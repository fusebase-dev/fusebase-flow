# Fusebase Flow v3.16.3 — token-trim (audit-driven; no semantics change)

**Release date:** 2026-06-10
**Previous version:** 3.16.2

## Summary

Closes the waste an independent token-economy audit found while answering "did FR-25 overcomplicate the framework for LLMs?" Audit verdict: **NET POSITIVE — with waste**. The gate clears break-even at 5-6 avoided monolith slice-reads per 100 sessions (actual coverage ~4-6×: one prevented 15k-line hot file saves ~40-60k tokens per touch vs ~243k/100 sessions of total FR-25 context cost). But ~33% of that cost was self-inflicted, and the audit's biggest find wasn't FR-25's at all.

## The headline find

`AGENTS.md` said "first action of every session: load `FLOW_RULES.md`" — unbounded. So the **amendment log** (~4.1k tokens, ~40% of the file, pure dated history) was loaded by every compliant session in every consumer repo: **~410k tokens per 100 sessions** paying zero operative instruction. The "dated history is free" assumption was false in practice.

## Audited savings

~**470k tokens / 100 compliant sessions / consumer repo** (410k amendment-log + 47k row dedupe + 12k preamble), recurring forever, zero behavior change. Deliberately NOT trimmed: CHANGELOG, release notes, change-notes, spec packs — genuinely cold once the read boundary exists.
