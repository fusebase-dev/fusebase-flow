# FuseBase Flow v3.29.0 — Codex / cross-agent slash-command parity

**Release date:** 2026-06-26
**Previous version:** 3.28.0
**Type:** MINOR (feature — additive)

## Summary

Until now Flow's **6 slash commands** (`/fusebase-health`, `/onboard`, `/product-owner`, `/handoff`, `/token-waste-audit`, `/find-wasted-effort`) were **Claude-Code-only** — the compatibility matrix told Codex/Cursor/Copilot/Gemini users to "invoke skills by name." This release ships **command parity** across every agent, in two arms:

- **B (primary, repo-portable).** A compact **6-row command-equivalents table** in `AGENTS.md` (and the propagated `CLAUDE.md` / `GEMINI.md`) — `Command · Claude /cmd · Codex /prompts:cmd (if installed) · Portable (invoke the skill / type the command)`. Repo-committed, every agent reads it, no native command mechanism required. This matches Codex's own recommended **skills** model.
- **A (optional, per-machine polish).** `hooks/local/install-codex-prompts.sh` — an explicit **opt-in** installer that single-sources native Codex custom prompts from the 6 canonical command bodies and writes them to `$CODEX_HOME/prompts/` (default `~/.codex/prompts/`), where Codex surfaces them as `/prompts:<cmd>`.

**FR-07-clean / additive:** no FR rule added (FR-01..FR-27 unchanged), no skill added (32 unchanged). No FR rule rows, no deploy-policy rule semantics, and no `ratchet-governance.yml` were touched. Independently **Codex-SHIP'd** (design RESCOPE folded — see below; implementation review LOW hardened → `cdf7bf6`, bite-verified).

## Ground truth (why the design rescoped)

The Codex design review supplied authoritative ground truth on its **own** mechanism (verified on Codex CLI **0.128.0**), which rescoped the original A-first plan:

- Native Codex custom prompts live in **`$CODEX_HOME/prompts/*.md`** (default `~/.codex/prompts/`) — **user-global, NOT repo-shared**, and **deprecated** by Codex in favor of skills.
- Invoked **`/prompts:<basename>`** (namespaced, not bare `/<name>`).
- They **use YAML frontmatter** (`description:`, `argument-hint:`) — do **not** strip it; args are `$1..$9` / `$ARGUMENTS`.
- A repo-local `.codex/prompts/` is **not** a Codex read path.
- Codex **plugins** bundle skills/mcp/apps/hooks, **not** slash commands.

Net: **B** is the genuine, repo-portable parity (and matches Codex's skills model); **A** is at best optional per-machine, namespaced, deprecated polish — honestly labeled as such.
