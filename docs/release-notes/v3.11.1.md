# FuseBase Flow v3.11.1 — sync-version-strings nested-docs prune (+ FLOW_RULES v0.9)

**Release date:** 2026-06-06
**Previous version:** 3.11.0

## Summary

A lightweight patch. `sync-version-strings.sh` rewrites live attestation strings (version / FR-range / skill-count) across framework files while **never** touching dated history (handoffs, specs, release-notes). Its prune list used exact top-level `-path` patterns (`./docs/handoff`, `./docs/specs`, `./docs/release-notes`, `./docs/fusebase-health`). `find`'s `-path` is exact — no implicit depth — so **per-app layouts** (`docs/<app>/handoff`, `docs/<app>/specs`, …) were not pruned, and the live-string rewrite reached into dated handoff/spec/release-note records and falsified their historical attestation versions. The script header (lines 34–39) states the explicit intent to never touch dated history; this restores that guarantee. Reproduced by the Product Owner before the fix.
