# Fusebase Flow v4.2.1 — adversarial-review hardening (test rigor + skill build-correctness)

**Released:** 2026-07-09
**Type:** patch (hardening + skill hygiene; no new capabilities)

A multi-model adversarial pass — Codex `gpt-5.6-sol` (xhigh) over the v4.2.0 implementation + CI fixes, and Fable 5 rating the 32 `flow-skills` on build-correctness-vs-purpose — surfaced tests that were green but did not pin their contract, plus skill matcher/body inconsistencies. Every finding was verified (some review claims were themselves refuted), fixed, and re-reviewed; each tightened test is **regression-proven** (reverting the code it guards makes the test fail).

## Test rigor / release-gate integrity

- **`run-tests.sh` exits boolean.** `exit $fail` truncated modulo 256, so exactly 256 failures exited 0 — and CI judges the step by exit code, so the release gate could go green with 256 real failures. Now `fail>0 ⇒ exit 1` (the numeric `N/N PASS` summary is unchanged).
- **Fixture coverage cannot silently vanish.** `run_hook_tests.py` fails (not silently skips) malformed-JSON / missing-`_handler` fixtures, enforces an expected-fixture floor (21) in both normal and `--compare-subprocess` modes, and fails `--compare-subprocess` on an in-process handler crash **before** the parity check (previously N identical crashes reported "parity 21/21 identical" and exited 0).
- **`--run-hook-tests` deep diagnostic** no longer reads a crash-after-PASS as OK: the full path requires `rc==0` with a strict `N/N PASS`; the MSYS fast path flags any nonzero rc and treats empty `0/0` output as broken. Locked by new platform-independent unit tests (`mv_deeprun_unit_*`).
- **`test-hook-manifest.sh`** now asserts the covered-set contract (`.jsonl` included; `*.local.*`/`__pycache__` excluded; deleted listed file ⇒ DRIFT). **`test-bootstrap-exception.sh`** scenario 8 asserts the masked git lists the staged file, the python3-absent path is non-blocking, and matches the exact non-enforcement warning string.

## Skill build-correctness (Fable 5 catalog rating: avg 90/100)

The catalog rated well-built and not template-theater; the residual defects were retrofit-hygiene at the edges, now closed:

- **`client-vs-internal`** — the "operator explicitly asks" trigger is now reachable end-to-end (skill body, `requirements-specification` consumer gate, `README`, `templates/audience.md`); worked example added.
- **`communication`** — description + anti-pattern reflect the v2.9.0 `references/patterns.md` lazy-load (no longer claim to contain the pattern library).
- **Lock/spec-status vocabulary** — `code-review`, `implementation-planning`, `task-delegation`, `requirements-specification`, `communication`, and `templates/spec.md` now use the real `decisions.md` convention (`Lock status: LOCKED`); the fictional `Locked: yes` marker and the always-true "Spec is DRAFT" negative-trigger are gone. `BLOCKED`/`ABORTED` retained as valid statuses.
- **`business-logic-guardian`** — the mandated 4-line impact-verdict block is emitted after the code check; worked example added.
- **`fusebase-flow-health-check`** — `PARTIAL_UNVERIFIED` added to the description's verdict list; `risk_level` normalized to the enum.
- **`comment-policy`** — the FR-22 cross-reference de-brittled.
