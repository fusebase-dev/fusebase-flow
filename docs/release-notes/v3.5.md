# Fusebase Flow v3.5.0 — input-dependent skills (client-facing delivery)

**Release date:** 2026-05-31
**Previous version:** 3.4.0

## Summary

v3.5.0 ships the remaining input-dependent skills from the gap analysis, all reusing the artifact-gated pattern proven by North Star in v3.4.0: complete skills that stay dormant until the project provides their data, so a fresh install is unaffected.

## Added

- **`client-vs-internal` skill** — applies the right posture per surface (client-facing = simple/guided/trust-first; internal = robust controls). Gated on `docs/audience.md`.
- **`product-docs-first` skill** — design per-app product docs before code, ingesting research + the North Star. Gated on `docs/<app>/product.md` / explicit request.
- **`business-logic-guardian` skill** — protect documented business logic during fixes (pairs with FR-20 zoom-out). Gated on `docs/<app>/business-logic.md`.
- **`product-apps-decomposition` skill** — "a product is composed of focused apps" guidance (reliability + token economy). Generic-with-enhancement: generic advice always, steers to the breakdown in `docs/<app>/product.md` when present.
- **`templates/audience.md`** and **`templates/product.md`** scaffolds.
- `session_start.py` scan extended to surface `docs/audience.md` and `docs/*/business-logic.md`.

## Pattern (unchanged from v3.4.0)

Every skill: existence-guard first step → absent artifact = silent no-op, never auto-create. Discovery is universal (AGENTS.md instruction + session_start scan + per-skill guard). `project-onboarding` may offer these artifacts (offer-once), never auto-creates them.

## Backlog status

This completes the input-dependent skill batch (G5/G6/G9/G11). Combined with v3.3.0 (generic skills + FR-20) and v3.4.0 (onboarding keystone + North Star), all Tier-1/Tier-2 gaps from the FuseBase positioning analysis are now shipped.

## Absent-by-default guarantee

A fresh install has none of `docs/audience.md`, `docs/<app>/product.md`, `docs/<app>/business-logic.md`. With them absent, all four skills are no-ops (G11 gives generic guidance) and Flow runs as a generic install. Run `/onboard` to opt in.

## Compatibility

```bash
bash hooks/local/mirror-skills.sh
bash hooks/local/preflight.sh
```
