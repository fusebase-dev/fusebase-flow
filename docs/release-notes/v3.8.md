# Fusebase Flow v3.8.0 — upgrade-path hardening 2

**Release date:** 2026-06-01
**Previous version:** 3.7.0

## Summary

A downstream ran the **real in-place 3.5.2 → 3.7.0 upgrade** on a live, heavily-customized install. The good news: **F2 (marker-anchored overlay refresh), F3 (opt-in hook wiring), and F4 (single-provider benign-absence) held up in practice** — health noise dropped from 23 findings to 2, settings.json was left untouched, and the legacy CLAUDE.md migrated correctly. The upgrade also surfaced **8 upgrade-path gaps** — one data-loss, one functional-staleness, plus consistency/pollution/UX. v3.8.0 fixes all 8. No skills added or removed (still 24 canonical).

## Fixed (severity-ranked)

### U1 (High — data loss) · overlay refresh no longer wipes operator values

The `### Project-specific values` table lives inside the marker-wrapped overlay block and the framework *invites* filling it — but a full-block refresh overwrote operator edits back to placeholders (the operator lost Project name + Stack set via `/onboard`). Fixed with an inner **`FLOW:PRESERVE`** sub-region:

- `agents-md-overlay.md` wraps the project-values table in `<!-- FLOW:PRESERVE:BEGIN -->…<!-- FLOW:PRESERVE:END -->`.
- `refresh_overlay_block()` builds an *effective template* = the fresh template with its preserve-region swapped for whatever the live block currently has (only when the operator actually customized it — otherwise the raw template, keeping the v3.7.0 byte-exact lock). Framework prose updates; operator values are carried forward verbatim.
- Recovery sim now asserts: customize a value, drift the framework prose, refresh → the value survives **and** the prose updates.

### U2 (High) · `upgrade.sh` now refreshes `hooks/`

It refreshed `skills/agents/workflows/policies/templates` but **not** `hooks/`. So a downstream that ran only `upgrade.sh` got FR-21 + new skills but a **stale hook layer** — the v3.7.0 tier-aware deploy gate silently didn't work, and the upgrade tooling (which lives in `hooks/local/`) couldn't update its own home. `hooks/` is now in the refreshed content set. `hooks/local/*.local.*` operator overrides are preserved; CLI-owned `.claude/hooks/**` (a separate tree) is untouched; the engine scripts self-update (the in-memory run finishes on the old engine; new logic takes effect next run).

### U3 (Med) · adapters stay internally consistent (FR-range + skill count)

`sync-version-strings.sh` bumped only the version, so an adapter without an overlay-refresh path — notably **GEMINI.md** — ended up self-attesting "v3.7.0 … FR-01 through FR-20" (the exact line a Gemini-CLI agent reads). Generalized to sync **derived attestation facts**: version **+** the FR-range (`FR-01..FR-NN`, derived from `FLOW_RULES.md`) **+** the `(NN canonical skills total)` count, across all adapters. Context-anchored (historical refs preserved); excludes dated dirs; idempotent.

### U4 (Med) · `upgrade.sh` stops polluting the consumer `docs/`

It copied 14 framework dev-docs (`architecture-overview.md`, `framework.md`, `constitution.md`, …) into the consumer's `docs/` root, colliding with consumer doc conventions. Now those are **not** copied by default; `--with-framework-docs` stages them under `docs/_fusebase-flow/` (namespaced).

### U5 (Med) · pre-3.6.0 bootstrap

`upgrade.sh` ships *inside* the version you need it to reach, so a pre-3.6.0 install couldn't use it. New **`hooks/local/bootstrap-upgrade.sh`** stages a source clone, copies the engine scripts into `hooks/local/`, then hands off to `upgrade.sh` (flags after `--`). The README documents a copy-paste one-liner for installs that lack even the bootstrap.

### U6 (Low) · LL ledger is opt-in / path-configurable

The durable tier record is `change_tier` + SHA in the commit body (git carries it). The consolidated `docs/changes/index.md` is now **opt-in** with a configurable path — a per-app docs layout may point it elsewhere or skip it. The `lightweight-lane` skill + `change-note` template no longer assume a repo-root ledger.

### U7 (Low) · legacy CLAUDE.md migration no longer doubles `---`

Migrating a pre-3.6.0 marker-less block kept the old pre-heading `---` while the template added its own. The begin-line-0 rebuild now also trims a trailing `---` rule from the preserved region, so exactly one separator remains. The marker-wrapped byte-exact lock (v3.7.0 F2) still holds.

### U8 (Low) · null-byte warning silenced

`sync-version-strings.sh` strips null bytes from scanned input (`tr -d '\0'`).

## Upgrading

```bash
# 3.6.0+ installs:
bash hooks/local/upgrade.sh --dry-run     # preview (now includes hooks/; framework docs NOT copied)
bash hooks/local/upgrade.sh               # apply (your FLOW:PRESERVE project values are kept)

# pre-3.6.0 installs (no upgrade.sh yet):
bash hooks/local/bootstrap-upgrade.sh --dry-run
```
