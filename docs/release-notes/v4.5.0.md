# Fusebase Flow v4.5.0

## `/find-wasted-code` — static friction-footgun audit

The third audit sibling, on a new axis:

| | `/find-wasted-code` (new) | `/find-wasted-effort` | `/token-waste-audit` |
|---|---|---|---|
| Axis | code-per-friction (footguns) | process-per-outcome (ceremony) | tokens-per-rule (consumption) |
| Inputs | repo source (docs, skills, commands, hooks, settings) | Flow artifacts on disk | Claude transcripts |
| Output | `docs/wasted-code/report.md` (tracked) | `state/audit/<date>.md` (gitignored) | `state/audit/<date>.md` (gitignored) |
| Trigger | `/find-wasted-code` ONLY (never auto) | `/find-wasted-effort` | `/token-waste-audit` |

It is the **active, static inverse of a papercuts log**: rather than relying on an
agent to notice friction and file it, it scans the repo for the same north-star
footguns and writes them down — dead-end tool calls, broken links, missing
helpers, footgun configs, and a silent-push-through baseline.

### Rules

| # | Rule | Confirms `broken` when… |
|---|---|---|
| W1 | Dead-end tool/script references | a root-explicit interpreter/`./`-run path is absent (execution context only — not `touch`/`rm` args) |
| W2 | Broken internal links | a link path is absent, or an anchor no heading yields (target slug-confident) |
| W3 | Missing helpers | a full-statement shell `source` of a static root-explicit script is absent |
| W4 | Footgun config | a settings hook `command` is wired to a handler that does not exist (silently never fires) |
| W5 | Silent push-through | — never confirmed; emits a measured swallowed-error **baseline** for review |

**Conservative by construction:** a finding is `confirmed` only when provable from
repository state; every ambiguity (cwd-relative paths, unparseable shell/markdown
context, ambiguous fields) routes to Coverage, never a false positive. Findings
are review candidates, never auto-fixes.

### How to run

```bash
python hooks/local/find-wasted-code.py            # writes docs/wasted-code/report.md
python hooks/local/find-wasted-code.py --print    # summary only, no write
python hooks/local/find-wasted-code.py --selftest  # 102 golden-fixture assertions
```

On non-Claude surfaces, invoke the `find-wasted-code` skill by name (it is
stdlib-only and surface-independent).

### Safety + isolation

- Manual-trigger only: `disable-model-invocation: true` (hard guarantee on Claude
  Code) + no lifecycle hook references the analyzer.
- stdlib-only, deterministic; inventory via `git ls-files` (host-independent,
  exact-case, Windows-safe).
- Read-only except one report write: containment-checked, symlink/hardlink-refusing,
  atomic (`mkstemp`), sentinel-guarded (won't overwrite a hand-authored file).
- The tracked report is redaction-safe — verified in `--selftest` against the
  repo's own secret scanner — so it never trips the pre-commit secret scan.

### Verification

Reviewed adversarially across six Codex-xHigh rounds (~21 defects found and fixed,
each locked with a regression fixture — the selftest grew from 56 to 102
assertions). Dogfooded on this repository: **0 false positives across 312 files**.
Known limitation (by design): a backtick code span across multiple physical lines,
or a `source` quoted across lines, is handled heuristically — these do not occur
in the repo's own files and degrade to false negatives, never false confirms
(see `flow-skills/find-wasted-code/references/rule-catalog.md`).

overlays, README + docs counts, both skill mirrors + manifest).
