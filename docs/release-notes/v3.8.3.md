# Fusebase Flow v3.8.3 — hooks-off ≠ drift (U11) + don't-delete-skills/ guard (U12)

**Release date:** 2026-06-01
**Previous version:** 3.8.2

## Summary

Two downstream findings. **U11** removes the last by-design-state-flagged-as-drift case (deliberate hooks-off). **U12** gets ahead of a FuseBase CLI deprecation that would otherwise break every Flow downstream: the CLI now warns that root `./skills` is "obsolete," but for a Flow install that folder is the canonical source of truth. This release ships the safe guard + docs and leaves the larger architectural realignment as an open, CLI-team-coordinated decision.

## Fixed

### U11 — Flow lifecycle hooks off is benign, not `SHARED_MERGE_DRIFT`

Hook wiring is opt-in (F3, v3.6.0). A `.claude/settings.json` that exists (CLI hooks present) but doesn't wire Flow's `stop.py` was reported as `SHARED_MERGE_DRIFT` — flagging a deliberate choice as drift. Now it reads as a **benign INFO** ("Flow lifecycle hooks not wired — opt-in; enable with `bash hooks/local/post-fusebase-update.sh --wire-hooks`"). A Flow merge that **clobbered** existing CLI Stop hooks is still a genuine `SHARED_MERGE_DRIFT`. Same shape as F4 / U10.

### U12 — guard against the CLI's `./skills` deprecation

Recent FuseBase CLI versions print `⚠️ The ./skills folder is obsolete and should be deleted` and ship skills under `.claude/skills/` only. That targets **non-Flow** projects. For a Flow install, root `skills/` is the **canonical source** that `hooks/local/mirror-skills.sh`, `hooks/local/upgrade.sh`, and the health mirror-count all build on. A downstream that follows the CLI's advice and deletes it would silently break Flow's next `upgrade.sh` (nothing to mirror) and the health mirror-count — and `fusebase update` can't restore it.

This release ships the **minimum safe guard** (the "document + guard" option from the report), which is correct under any future end-state and forecloses none:

- **Health check:** `check-cli-flow-conflicts.sh` now flags an empty/absent root `skills/` *while Flow mirrors still exist* as a loud, recoverable `FLOW_LAYER_DRIFT` — "do NOT delete it; the CLI 'obsolete ./skills' warning does not apply to Flow installs; restore with `bash hooks/local/upgrade.sh` (or `bootstrap-upgrade.sh`, or `git checkout -- skills/`)."
- **Docs:** the AGENTS.md overlay "Maintenance posture" section (so every downstream's AGENTS.md carries it) and the README health section document the do-not-delete / ignore-the-CLI-warning guidance.

**Deliberately deferred (open decision):** the larger realignment — move Flow's canonical store off root `skills/` to a Flow-namespaced path the CLI won't touch, *or* mirror into `.claude/skills/` as the source-of-truth (aligning with the CLI's new model) — is **not** done here. It changes Flow's whole source→mirror spine and the right end-state depends on coordinating with the CLI team on where `skills/` is going. The guard above buys time without committing to a direction.

## Notes for downstreams

- If your health check showed `SHARED_MERGE_DRIFT` only because you deliberately keep Flow hooks off, that now reads benign.
- **Do not** act on the CLI's "delete ./skills" warning while Flow is installed. If you already did, restore with `bash hooks/local/upgrade.sh` (or `git checkout -- skills/`).
