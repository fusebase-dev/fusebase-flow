# Fusebase Flow v3.16.4 — efficiency repairs (two broken consumer paths + drift sweep)

**Release date:** 2026-06-10
**Previous version:** 3.16.3

## Summary

Repair pass from a framework-wide independent efficiency audit. Two findings were **bugs**, not inefficiencies: the documented existing-repo install copied a directory retired in v3.9.0 (consumers got zero Flow skills), and the documented hook activation used a variable Claude Code never sets (all six Flow lifecycle hooks silently dead). No rule or behavior-model change.
