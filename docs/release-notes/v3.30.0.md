# v3.30.0 — FuseBase CLI 0.25.9 vendor refresh + de-staled CLI-Stop-hook model

**Date:** 2026-06-29
**Type:** MINOR (hook-merge behavior change + new vendored provider skill)
**Baseline:** FuseBase Flow v3.29.0 → v3.30.0

## Summary

Flow v3.29.0 already **ran** with FuseBase CLI 0.25.9 (structurally compatible — same AGENTS.md / CLAUDE.md / `.claude/{skills,agents,hooks}` / settings.json conventions; restore never clobbers CLI assets), but two surfaces were **stale** against 0.25.9 and produced real warts:

1. **A proven false `SHARED_MERGE_DRIFT`.** Flow hardcoded the CLI's Stop-hook set as `[run-typecheck-apps.js, quality-check-apps.js]`. CLI 0.25.9 replaced the wired `run-typecheck-apps.js` with `run-typecheck-on-stop.sh` and added `run-lint-on-stop.sh`, wiring `run-typecheck-apps.js` **0 times**. On a 0.25.9 project the health check flagged a phantom "CLI Stop hooks not preserved: run-typecheck-apps.js."
2. **A redundant double-typecheck on restore.** `settings-json-merge.py` re-injected `run-typecheck-apps.js` from a static name → typecheck ran twice on Stop (`run-typecheck-apps.js` + `run-typecheck-on-stop.sh`).

Nothing was broken — Flow installed and worked on 0.25.9. This release removes the noise, the redundant typecheck, and the false positive, and future-proofs the model.

## What changed

### B (load-bearing) — de-staled CLI-Stop-hook model (D1, diff-framed, version-agnostic)
- **`settings-json-merge.py` is now preserve-only.** It appends Flow's `stop.py` once (idempotent) and preserves every existing Stop hook + `enabledMcpjsonServers`; it no longer re-injects `run-typecheck-apps.js` from a static name. It never removes an existing hook, so an older-CLI project that still wires `run-typecheck-apps.js` keeps it.
- **`check-cli-flow-conflicts.sh` now diff-frames `SHARED_MERGE_DRIFT`.** It flags drift only when Flow's merge actually **dropped** a CLI Stop command that was wired in the pre-merge `.claude/settings.json` Stop chain (source = the project's real settings, not static names; `.claude/hooks/` is used only to classify an already-wired command as CLI-owned). No false positive on 0.25.9's 3-hook set; a genuinely-dropped hook is still flagged.

### H1 — `.claude/settings.json.example`
- Stop chain aligned to 0.25.9's wired set: `[run-lint-on-stop.sh, run-typecheck-on-stop.sh, quality-check-apps.js, stop.py]`. No `run-typecheck-apps.js`. The "deprecated" comments on the `*-on-stop.sh` hooks were removed. `stop.py` stays discoverable to `settings-json-merge.py`.

### A (freshness) — full re-vendor to FuseBase CLI 0.25.9
- Re-copied all CLI-owned assets over Flow's vendored copies: **20** provider skills (+ `references/`; adds the new `app-api-contract-testing`; none removed/renamed), **4** `.claude/hooks` (adds `run-lint-on-stop.sh`, `run-typecheck-on-stop.sh`), and **2** app-agents across `.claude/skills`, `.agents/skills`, `.claude/hooks`, `.claude/agents`, `.codex/agents`. Re-ran `stamp-cli-provenance.sh` → fresh `audit/cli-vendor-manifest.json` (62/62 byte-match to the real 0.25.9 CLI; no `CLI_SNAPSHOT_STALE`).

### H2 — flag-gate the new skill
