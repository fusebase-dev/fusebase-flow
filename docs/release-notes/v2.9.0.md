**Released:** 2026-05-10

## TL;DR

Token-efficiency initiative. **6 coordinated changes** reduce per-session and per-ticket token cost by ~6000 + ~3000 tokens respectively, without losing any functional content. ## FR-18 — the supersede discipline

When a deploy aborts, an operator clarify forces a respec, or a gate run fails and re-runs, the corrected version REPLACES the stale content. Don't preserve both. Git history is the audit trail.

### Exception — when human-readable diff is essential

```markdown
## Superseded sections (audit only — agents skip)

> The content below was the original v1 plan. Superseded 2026-05-10 by the
> current handoff body. Kept inline (rather than via git history) because
> the diff itself is part of the ticket's audit trail. Per FR-18, agents
> reading this file should skip this section — it is not authoritative
> for current state.

<old content here>
```

The heading is a structural marker:
- **Humans** see the section for forensic review.
- **Agents** recognize the heading and skip the section's body when extracting authoritative content.

Use sparingly. The default is REPLACE.
