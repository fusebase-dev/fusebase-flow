# Fusebase Flow v3.18.0 — integration-debloat (procedure layer right-sized)

**Release date:** 2026-06-10
**Previous version:** 3.17.1

## Summary

A capability-integration audit (functional-overlap + velocity lens across all 10 capability groups) found the enforcement layer sound but the **procedure layer violating the framework's own pointer rules**: the gate contract in 7 carriers, smoke in 6, 130 lines of legacy snippets in always-read workflows, a duplicated review pass, 3 live cross-surface contradictions, and dead machinery. This release fixes all of it — independent implementer against a locked spec (`docs/specs/integration-debloat/spec.md`, G1..G11), PO-verified, no rule-text change.

## De-duplication (−9,305 bytes / −20.3% across per-ticket-read files)

- Gate-report contract → exactly 2 canonical carriers (`policies/gate-contracts.yml` machine + `templates/gate-report.md` producer); 7 other restating surfaces → pointers.
- Smoke contract → canonical `flow-skills/smoke-testing`; `smoke-verification.md` 5,715→3,349 B (mechanics only); validation-and-qa sub-mode B → 3 lines. Stop-hook fixture semantics untouched (13/14/15/16 green).
- Legacy handoff snippets deleted from both greenlight workflows (templates canonical since v2.5.0). `greenlight-implement.md` 8,633→5,026 B.
- Review boundary: `code-review` trusts the recorded gate verdict for deterministic fields (AC↔task, lint/typecheck, TODO, protected paths) — semantic dimensions only; one full duplicated diff pass per ticket eliminated.

## Deploy ceremony right-sized (the "painful middle tier")

**Reversible-deploy waiver:** deploy handoffs now carry `dp1_waiver: eligible|excluded`. On *eligible* tickets (reversible; no protected-path/security/migration surface) the operator's typed `APPROVE-DEPLOY-NOW` (DP.6) also authorizes the Deploy agent to stamp the DP.1 artifact itself (`approve-local.sh`). The artifact stays mandatory and on-disk — FR-12 and hook semantics unchanged; the human-at-keyboard gate unchanged; deploy-intent confirmations drop 3 → 2. Excluded classes keep operator-run DP.1. (Security note in the spec: worst case equals today's behavioral baseline.)
