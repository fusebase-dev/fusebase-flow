# Fusebase Flow v3.20.0 — FR-26 token-efficient execution + /token-waste-audit

**Release date:** 2026-06-11
**Previous version:** 3.19.1

## Summary

Operator-driven: implementation sessions leak tokens when nobody is watching the spend — and nobody could watch it, because there was no measurement. v3.20.0 closes the last uncovered leak class with a **discipline + audit** pair, designed around one hard constraint: *must not damage product quality, development depth, or LLM intelligence*. Built under a double-review protocol: independent **plan review** (caught a real blocker — transcript usage lines repeat per request; naive summation overcounts ~2.4×), spec self-correction, implementation, independent **implementation review** (2 count fixes pre-ship).
