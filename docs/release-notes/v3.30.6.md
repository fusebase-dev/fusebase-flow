# FuseBase Flow v3.30.6 — gate wall-time optimization (FF_ONLY scoped gates + preflight batch + adaptive reap poll)

**Release date:** 2026-07-03

## Summary

Cuts gate wall-time without touching any runtime behavior. Phase B of the operator's optimization mandate: on the saturated host the gate is ~90% of session wall time, so this release scopes and batches the gate machinery while preserving coverage and every fail-closed guarantee. No new FR rule (FR-01..FR-27 unchanged) and no new Flow skill (32 unchanged). The v3.30.5 fail-closed FR-07 §3 / FR-12 §2 pre-commit chain (`hooks/git/pre-commit` + `hooks/shared/**`) is **UNTOUCHED**.

Independently reviewed (Codex + a 3-lens Opus panel): the optimizations are sound (the reap-loop deadline floor is preserved with no early reap; `FF_ONLY` is fail-closed by construction; the preflight batch is byte-identical drift/missing detection), and the 13 full-gate FAILs are pre-existing host-saturation artifacts that reproduce identically on the baseline — NOT regressions. Additive / FR-07-clean: no FR rule rows, no deploy-policy rule semantics, no `ratchet-governance.yml` touched — only the FLOW_RULES version-attestation line moved.

## What ships

| Piece | What it does |
|---|---|
| **FF_ONLY opt-in scoped gates (F2)** | `run-tests.sh` gains `FF_ONLY` — run only named suites during the implement loop. Unset ⇒ byte-identical full gate. Scoped ⇒ fail-closed by construction: the scoped summary deliberately fails the strict `N/N PASS` classifier, writes a separate results file, and exits 2 on an unknown/empty tag — a scoped run can never be mistaken for a clean full pass. New `test-ff-only.sh` guards the contract. The final pre-commit / pre-deploy gate MUST still be a full unscoped run. |
| **preflight mirror-hash batching (F4)** | Preflight runs one `sha256sum` per root instead of ~270 per-file spawns (~6.7× faster preflight; repairs the default health-check preflight budget). Drift/missing detection is byte-identical. |
| **spawn micro-cuts (F5)** | Fixture hot-loop spawn cuts — builtin file reads + a metadata pre-pass replace per-fixture `$(echo|sed)`/python invocations. |
| **adaptive sub-second reap poll (F3)** | The MSYS bounded-run reap loop uses a spawn-free FIFO nap ladder with `EPOCHREALTIME` accounting — the deadline is preserved as a hard FLOOR (never reaps early), falling back verbatim to the prior `sleep 1` on any host without the builtin nap. |
