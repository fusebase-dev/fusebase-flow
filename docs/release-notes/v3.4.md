# Fusebase Flow v3.4.0 — onboarding keystone + North Star

**Release date:** 2026-05-31
**Previous version:** 3.3.0

## Summary

v3.4.0 ships the **onboarding keystone**: the mechanism that lets a project be optimized for its own vision, and the first **artifact-gated** skill (North Star) that proves the pattern. This unlocks the input-dependent skills — which are complete skills that stay dormant until the project provides their data, so a fresh install is unaffected (no clutter, no degradation).

## The model

- **Generic engine ships complete; project artifact is absent by default.** A skill like `north-star` is fully written but does nothing until `docs/north-star.md` exists.
- **Onboarding creates the artifacts.** `project-onboarding` (`/onboard`) interviews the operator and writes `docs/north-star.md` (+ fills AGENTS project-values). Operator-triggered and optional — skip it and Flow runs generically.
- **Discovery is universal (hook-independent).** Three layers so it works on every surface:
  1. **AGENTS.md "Active project context"** instruction — every agent (Claude/Codex/Cursor/Copilot/Gemini) reads it. Source of truth.
  2. **`session_start.py`** scan — Claude Code accelerator (additive, read-only). Never a dependency.
  3. **Per-skill existence-guard** — each input-dependent skill's first step checks its artifact; absent → silent no-op, never auto-create.

## Added

- **`project-onboarding` skill** (PO-owned) — discovery interview → writes `docs/north-star.md`, fills AGENTS project-values, ingests operator research. Re-runnable; creates nothing unless the operator provides content.
- **`north-star` skill** (artifact-gated) — reads `docs/north-star.md` if present and keeps all work on-vision; **silent no-op if absent**. The canonical artifact-gated pattern.
- **`/onboard`** and **`/product-owner`** slash commands (`.claude/commands/`).
- **`templates/north-star.md`** — the scaffold onboarding populates.
- **AGENTS.md "## Active project context — read first"** — universal discovery instruction.
- **`session_start.py`** — additive scan that surfaces project artifacts on Claude Code (absence is reported as "not onboarded", changes nothing).

## Out of scope (follow-up slice)

The remaining input-dependent skills — `client-vs-internal` (G5), `product-docs-first` (G6), `business-logic-guardian` (G9), `product = many small apps` (G11) — reuse the identical artifact-gated pattern proven here and ship next. `docs/constitution.md` stays Flow-owned (project vision lives in `docs/north-star.md`). Checksum-based change-detection deferred (presence + `last_updated` only).

## Absent-by-default guarantee

A fresh install has **no** `docs/north-star.md`. With it absent: `north-star` is a no-op, `session_start.py` reports "not onboarded", and Flow behaves exactly as a generic install. Run `/onboard` to opt in.

## Compatibility

```bash
bash hooks/local/mirror-skills.sh
bash hooks/local/preflight.sh
bash hooks/tests/run-tests.sh
```

No per-ticket artifact migration required.
