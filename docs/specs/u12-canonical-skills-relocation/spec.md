# Spec — U12 canonical-skills relocation (Issue-1 end-state #2)

**Status:** DONE (shipped v3.9.0)
**Lane:** Full (high blast radius — touches Flow's mirror/upgrade/health spine)
**Target version:** 3.9.0
**Supersedes:** the v3.8.3 guard (kept as a transitional safety net during migration)

## Problem

The FuseBase CLI deprecates the root `./skills` folder (`⚠️ The ./skills folder is obsolete and should be deleted`). Flow has used root `skills/` as its **canonical** source of truth (read by `mirror-skills.sh`, `upgrade.sh`, both health engines, `preflight.sh`, `session_start.py`). The collision means:

- Operators who follow the CLI warning delete Flow's canonical source and silently break the mirror/upgrade/health model (mitigated today by the v3.8.3 guard, which is warn-only on the CLI side).
- The guard is non-foreclosing but leaves a standing collision; a future CLI escalation (active deletion on `fusebase update`) would break every Flow install.

## Goal

Move Flow's canonical skills out of the CLI-claimed `skills/` name to a Flow-namespaced path the CLI never touches, so the collision cannot exist under any CLI behavior — with **zero-touch migration** for existing downstream installs.

## Scope

- **In:** relocate canonical `skills/` → `flow-skills/`; repoint every reader; auto-migrate existing installs on `upgrade.sh`; invert the U12 guard; update CI allowlist, overlays, root docs, install doc; extend the recovery sim.
- **Out:** `agents/` (not deprecated by the CLI — stays at root). Provider mirror layout (`.claude/skills/`, `.agents/skills/`) is unchanged — those are generated by name and the CLI owns `.claude/skills/`.

## Non-negotiables

1. **No data loss / no broken install on upgrade.** A downstream on root `skills/` must end with canonical at `flow-skills/`, root `skills/` removed, mirrors intact, health HEALTHY — in one `upgrade.sh` pass.
2. **Legacy fallback.** Every reader prefers `flow-skills/` but falls back to legacy `skills/` so a half-migrated tree still functions.
3. **Provider mirrors unchanged.** `.claude/skills/<name>/SKILL.md` and `.agents/skills/<name>/SKILL.md` keep their paths (names unchanged); plugin manifest unaffected.
4. **Precision retained.** A genuinely deleted canonical (now `flow-skills/`) while mirrors exist still surfaces loudly + recoverably.

## Acceptance

- `flow-skills/` holds the 24 canonical skills; root `skills/` gone from the repo.
- preflight 0/0; run-tests all PASS; recovery sim PASS incl. a new migration test; health HEALTHY (24 skills); mirror drift 0; plugin validate clean.
- CI public-surface allowlist accepts `flow-skills`, rejects a stray `skills` reappearing.
