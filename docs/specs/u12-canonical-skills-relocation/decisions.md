# Decisions — U12 canonical-skills relocation

## D1 — New canonical location: `flow-skills/` (NOT `.fusebase-flow/skills/`)

**Locked with operator as A1 (`.fusebase-flow/skills/`), changed to A2 (`flow-skills/`) on a hard blocker.**

`.fusebase-flow/` is **already gitignored** (`.gitignore:53`, a defensive runtime-state catch — hook handlers with a wrong cwd once wrote `.fusebase-flow/state/audit.log.jsonl` there). Tracking canonical source inside a gitignored runtime-state namespace would need a fragile `!`-exception and is a footgun (clearing runtime state could nuke canonical). `flow-skills/` is a clean, visible, tracked top-level dir; the CLI deprecation targets the literal `skills` name only, so `flow-skills/` is collision-free. This was the operator's offered alternative (Decision A option 2).

## D2 — Scope: skills only

`agents/` is **not** deprecated by the CLI — it stays at root. Moving it would double the blast radius for no current benefit. (Operator-locked B1.)

## D3 — Migration is automatic + idempotent, with legacy fallback

`upgrade.sh` (and `bootstrap-upgrade.sh` via it) lands canonical at `flow-skills/` from the source, then removes a legacy root `skills/` (backed up `.pre-upgrade-<ts>`). Every reader prefers `flow-skills/` and falls back to legacy `skills/`, so a partially-migrated or pre-upgrade tree still works. Re-running is a no-op.

## D4 — U12 guard inverts (not deleted)

Old guard: "root `skills/` is canonical — do NOT delete it." New guard: canonical is `flow-skills/`; if `flow-skills/` is absent while mirrors exist → loud, recoverable MISSING. A leftover root `skills/` is now benign (the CLI's "delete ./skills" warning is finally correct for Flow too) — surfaced as a one-line INFO advising migration via `upgrade.sh`.

## D5 — Provider mirrors + plugin unchanged

`.claude/skills/` and `.agents/skills/` keep their paths (generated by skill name). The CLI still owns `.claude/skills/`; Flow still regenerates it from canonical. `.claude-plugin/plugin.json` references `.claude/skills` — unaffected.

## D6 — Version: 3.9.0

Structural change (canonical layout moves; public top-level surface changes), zero behavior loss → minor bump, not patch.
