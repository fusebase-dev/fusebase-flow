# FuseBase Flow

**面向客户的团队用来借助 AI 构建内部应用和面向客户应用的框架。**

> 🌐 这是翻译后的摘要。完整且始终最新的文档请见 **[英文 README](../../../README.md)**（权威版本）。

FuseBase Flow 为面向客户的团队提供单一联系人——一个 **Product Owner**（产品负责人）智能体，就构建什么以及如何构建提供咨询，然后通过 **AI Developer**（AI 开发者）智能体将其交付为可上线的应用。它通过仓库文件来引导你现有的 AI 智能体（无需 SaaS），并可在 Claude Code、Codex、Cursor、Copilot 和 Gemini 中使用。

## 适用对象

- 为自身运营和客户构建应用的代理机构、咨询公司和面向客户的团队。
- 希望将自身专业能力产品化为应用和系统的团队。

## 两个智能体

- **Product Owner** — 你的单一联系人：提供咨询、给出建议，并将工作拆分为各个阶段和切片（明确需求 → 澄清 → 计划 → 决策 → 任务 → 评审 → 部署交接）。
- **AI Developer** — 按"一个任务 = 一次提交"进行构建，在验证关卡处停下，仅在获得批准后才部署。

## 快速开始

1. 在 GitHub 上点击 **"Use this template"** 并创建你的仓库。
2. 在你的 AI 工具中打开它——它会读取 `AGENTS.md` / `CLAUDE.md` 并遵循 `FLOW_RULES.md`。
3. 对智能体说：*"我们来交付 `<功能>`。"* — 或先运行 `/onboard` 进行定制以适配你的团队。

完整说明、技能目录、常见问题和安装方式都在 **[英文 README](../../../README.md)** 中。
