# FuseBase Flow

**クライアント対応チームが、社内向け・クライアント向けアプリを AI で構築するためのフレームワーク。**

> 🌐 これは翻訳された要約です。完全で常に最新のドキュメントは **[英語版 README](../../../README.md)**（正式かつ権威ある版）にあります。

FuseBase Flow は、クライアント対応チームに単一の窓口を提供します。何をどのように構築するかを助言する **Product Owner** エージェントが要件をまとめ、それを **AI Developer** エージェントが実装してアプリとして出荷します。リポジトリ内のファイルを通じて既存の AI エージェントを方向づけ（SaaS 不要）、Claude Code、Codex、Cursor、Copilot、Gemini で動作します。

## 対象となるユーザー

- 自社の業務とクライアントの両方に向けてアプリを構築する、代理店・コンサルティング会社・クライアント対応チーム。
- 自分たちの専門知識をアプリやシステムとして製品化したいチーム。

## 2 つのエージェント

- **Product Owner** — あなたの単一の窓口。助言し、方向づけ、作業をフェーズとスライスに分解します（仕様化 → 明確化 → 計画 → 意思決定 → タスク → レビュー → デプロイ引き継ぎ）。
- **AI Developer** — 1 タスク = 1 コミットで実装し、検証ゲートで停止し、承認があった場合のみデプロイします。

## クイックスタート

1. GitHub で **「Use this template」** をクリックし、リポジトリを作成します。
2. お使いの AI ツールで開きます。`AGENTS.md` / `CLAUDE.md` を読み込み、`FLOW_RULES.md` に従います。
3. エージェントにこう伝えます: *「`<機能>` を出荷しましょう。」* — または先に `/onboard` を実行してチームに合わせてカスタマイズします。
