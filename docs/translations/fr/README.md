# FuseBase Flow

**Le framework que les équipes en contact avec les clients utilisent pour créer des applications internes et destinées aux clients avec l'IA.**

> 🌐 Ceci est un résumé traduit. La documentation complète et toujours à jour se trouve dans le **[README en anglais](../../../README.md)** (version de référence faisant autorité).

FuseBase Flow offre aux équipes en contact avec les clients un point de contact unique : un agent **Product Owner** qui conseille sur quoi construire et comment, puis le transforme en applications livrées grâce à un agent **AI Developer**. Il façonne votre agent IA existant via des fichiers du dépôt (sans SaaS) et fonctionne avec Claude Code, Codex, Cursor, Copilot et Gemini.

## À qui s'adresse-t-il

- Agences, cabinets de conseil et équipes en contact avec les clients qui créent des applications pour leurs propres opérations comme pour leurs clients.
- Équipes qui souhaitent transformer leur expertise en applications et systèmes.

## Les deux agents

- **Product Owner** — votre point de contact unique : il conseille, oriente et découpe le travail en phases et en lots (Spécifier → Clarifier → Planifier → Décisions → Tâches → revue → transfert pour déploiement).
- **AI Developer** — il construit une tâche = un commit, s'arrête à la porte de vérification et ne déploie qu'après approbation.

## Démarrage rapide

1. Cliquez sur **« Use this template »** sur GitHub et créez votre dépôt.
2. Ouvrez-le dans votre outil d'IA : il lira `AGENTS.md` / `CLAUDE.md` et suivra `FLOW_RULES.md`.
3. Dites à l'agent : *« Lançons `<fonctionnalité>`. »* — ou exécutez `/onboard` d'abord pour l'adapter à votre équipe.

Les instructions complètes, le catalogue des skills, la FAQ et les chemins d'installation se trouvent dans le **[README en anglais](../../../README.md)**.
