# QP — State & navigation (loaded on trigger match; see ../SKILL.md)

| ID | Trigger | Requirement | Verify (copy-ready smoke recipe) | Anti-pattern |
|---|---|---|---|---|
| QP-01 | Any view with filters, search, sort, pagination, tabs, date ranges, or selected report/flow | **The URL reflects the view state.** Every state parameter that changes what the user sees is encoded in the URL (query params or route); refreshing, sharing, or bookmarking the URL restores the exact same view. | Apply 2+ filters + a sort → copy URL → hard-refresh → assert identical view; open the URL in a fresh tab/session → assert same state (after auth). | State only in component memory/store; refresh resets to defaults; "share this report" sends a URL that opens the unfiltered default. |
| QP-02 | Multi-view/SPA navigation | **Browser back/forward work.** Back returns to the previous app state (not a blank page, not an exit); forward re-applies it. In-app navigation pushes history entries for state changes a user would expect to step back through. | From list → detail → edit, press back twice → assert list with prior state; press forward → assert detail again. | `history.replaceState` everywhere (back exits the app); modal/tab changes that hijack back unexpectedly. |
| QP-03 | Any entity addressable by URL (detail pages, deep links) | **Deep links are guarded.** A direct URL to a missing, deleted, or unauthorized entity renders a designed not-found / no-access state with a way forward — never a crash, infinite spinner, or blank screen. | Request a detail URL with a non-existent ID and (if applicable) another tenant's ID → assert designed 404/403 state, no console error, working "back to list" action. | Unhandled fetch rejection → white screen; showing an empty record shell as if it existed. |
| QP-04 | List → detail → back flows on long lists | **List position survives.** Returning from a detail view restores scroll position and page of the list (works with QP-01: page/cursor in URL). | Scroll to page 3 / row 50 → open a detail → back → assert same page + approximate scroll position. | Back always lands on page 1 top, forcing re-navigation. |
